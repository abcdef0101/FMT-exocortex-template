Выполни протокол Day Open для роли Стратег (R1).

## Режим: автономный тестовый прогон

> **Ты выполняешь Day Open в headless-режиме (неинтерактивно).**
> Пользователя нет. Все решения принимай самостоятельно по правилам ниже.
> Не задавай вопросов. Не жди подтверждения. Действуй.

## Контекст

- **WORKSPACE_DIR:** {{WORKSPACE_DIR}}

## Правила автономных решений

Применяй эти правила когда протокол требует решения, которое в интерактивном режиме принял бы пользователь:

1. **Календарь:** использовать тестовые блоки из вчерашнего DayPlan (09:00-11:00 занят, 11:00-13:00 свободен, 14:00-15:00 занят, 15:00-19:00 свободен). Не вызывать реальный календарь.
2. **GitHub Issues:** не вызывать `gh issue list`. Использовать `{{WORKSPACE_DIR}}/DS-strategy/inbox/seed-issues.md` как источник issues. Если файла нет — секция «Issues: нет данных (тестовый прогон)».
3. **Scout / IWE-здоровье / Мир / Видео:** пометить «тестовый прогон, пропущено». Не вызывать реальные проверки.
4. **Заметки (шаг 4):** автономная категоризация из `fleeting-notes.md`. Bold заметки → категория (Задача / Знание / НЭП / Черновик / Шум). 🔄 заметки → если >7 дней, предложить в план. ✅ processed → игнорировать.
5. **Budget Spread (шаг 5.2):** применять автономно: threshold 2h, rounding 0.5h. Для каждого РП с бюджетом ≥2h вычислить daily_slot = round(budget_week / days_left, 0.5).
6. **Саморазвитие (шаг 6):** тестовый слот «Чтение FPF» — 1h. Первый слот в плане дня.
7. **План дня (шаг 5):** формировать строго по приоритету входов: (a) carry-over из «Завтра начать с» вчерашнего DayPlan → BLOCKING, все без фильтрации, (b) WeekPlan — in_progress + pending РП с проверкой дедлайнов, (c) MEMORY.md кросс-проверка, (d) mandatory ежедневные РП из day-rhythm-config.yaml.
8. **Commit:** закоммитить созданный DayPlan в DS-strategy. `git add` + `git commit` + `git push`.

## Процесс (сокращённый 17-шаговый алгоритм)

Выполни шаги в порядке, помечая пропущенные:

### 0. Разрешить WORKSPACE_DIR и FMT_DIR
Определи пути.

### 1. Extensions (before) — ПРОПУСТИТЬ
Тестовый прогон, расширения не загружены.

### 2. Вчера
**2.1.** Прочитай вчерашний DayPlan (`DayPlan {{YESTERDAY}}.md`). Извлеки:
- Секцию «Итоги дня» → результаты (1-3 ключевых)
- Секцию «Завтра начать с» → **приоритетный вход** для шага 5
- Статус: есть ли секция «Итоги»? Если нет → ⚠ «День не был закрыт»

**2.2.** Коммиты за вчера: возьми из git log (файлы уже созданы seed-скриптом в WORKSPACE_DIR).

### 3. GitHub Issues — ЗАМЕНИТЬ
Не вызывать `gh`. Прочитать `seed-issues.md` в inbox/. Связать с РП по ключевым словам.

### 4. Заметки
Прочитай `fleeting-notes.md`. Категоризируй каждую bold-заметку:
- «Add auto-cleanup...» → Задача → предложить в DayPlan
- «deepseek/deepseek-chat API latency...» → Знание → capture в Pack
- 🔄 «Интеграция с Grafana...» → свежая (<7 дней) → оставить 🔄
- Обычные заметки → прочитать, отметить в «Разбор заметок»
- НЕ удалять заметки из файла.

### 5. План на сегодня
Строгий порядок входов:
1. **Carry-over из Day Close (БЛОКИРУЮЩЕЕ):** все РП из «Завтра начать с» → в план
2. **WeekPlan:** in_progress + pending РП, проверить дедлайны, применить Budget Spread
3. **MEMORY.md** → кросс-проверка пропущенных РП
4. **mandatory_daily_wps** из day-rhythm-config.yaml

### 6. Саморазвитие
Слот «Чтение FPF» — 1h. Первый слот (⚫).

### 7. Стратегирование — ПРОПУСТИТЬ
Сегодня вторник, не strategy_day.

### 8. Помидорки
Из day-rhythm-config.yaml: 25/5, длинный перерыв каждые 4 цикла. Упомянуть в DayPlan.

### 9. Календарь — ТЕСТОВЫЕ ДАННЫЕ
Использовать блоки: 09:00-11:00 занят, 11:00-13:00 свободен, 14:00-15:00 занят, 15:00-19:00 свободен.

### 10. IWE за ночь — ПРОПУСТИТЬ
Пометить: «тестовый прогон, пропущено».

### 11. Видео — ПРОПУСТИТЬ
`video.enabled: false`.

### 12. Контент — АВТО
Из Strategy.md + WeekPlan: 1-2 темы для публикаций. Например: «Golden image testing: от 15 мин до 14 сек».

### 13. Scout — ПРОПУСТИТЬ
Пометить: «тестовый прогон, пропущено».

### 14. Мир — ПРОПУСТИТЬ
`news.enabled: false`.

### 15. Требует внимания
Собрать из шагов 2-14. Должно включать:
- Факт что это тестовый прогон
- Если есть незакрытые вопросы из вчерашнего DayPlan
- Если seed-issues.md содержит открытые issues

### 16. Extensions (after) — ПРОПУСТИТЬ

### 17. Запись
**17a.** Записать DayPlan: `{{WORKSPACE_DIR}}/DS-strategy/current/DayPlan YYYY-MM-DD.md` по шаблону DayPlan (collapsible секции: План на сегодня, Календарь, IWE за ночь, Разбор заметок, Итоги вчера). Предыдущий DayPlan → `archive/day-plans/`.

**17b.** Commit + Push:
```bash
cd "{{WORKSPACE_DIR}}/DS-strategy"
git add current/ archive/
git commit -m "day-open: DayPlan $(date +%Y-%m-%d)"
git push
```

**Результат:** DayPlan с актуальным планом на сегодня, все секции заполнены, carry-over RP на первом месте, бюджет распределён.
