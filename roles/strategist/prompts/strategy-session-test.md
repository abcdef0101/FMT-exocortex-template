## Предусловие

> **Черновик WeekPlan уже создан** сценарием session-prep.
> Если черновика нет — создай его самостоятельно (упрощённо: прочитай итоги прошлой недели, собери РП из MEMORY.md, сформируй таблицу).

## Правила автономных решений

Применяй эти правила когда сценарий требует решения, которое в интерактивном режиме принял бы пользователь:

1. **Inbox:** 🔄 заметки >7 дней → в план. Остальное → архивировать.
2. **НЭП:** resolved → закрыть (обновить статус в Dissatisfactions.md). active → оставить.
3. **Carry-over:** перенести в новый WeekPlan без изменений (те же бюджет, приоритет).
4. **Бюджеты:** не менять. Использовать значения из предыдущего WeekPlan.
5. **Новые РП:** не добавлять (только перенос существующих).
6. **Утверждение:** автоматическое. `status: draft` → `status: confirmed`.
7. **Коммит:** закоммитить все изменения в DS-strategy.

## Процесс

#### 1. Открытие сессии

- Найди `WeekPlan W*.md` со `status: draft` в `DS-strategy/current/`
- Выведи краткую сводку: повестка, ключевые вопросы, carry-over
- Начинай по повестке автоматически

#### 2. Ревью прошлой недели

- Прочитай блок «Итоги прошлой недели» из черновика
- Проанализируй completion rate, carry-over, инсайты
- Зафиксируй выводы (автономно)

#### 3. Разбор inbox

- Прочитай `DS-strategy/inbox/fleeting-notes.md`
- Примени правила автономных решений (см. выше)
- Удали/архивируй обработанные заметки

#### 4. Неудовлетворённости (НЭП)

- Прочитай `DS-strategy/docs/Dissatisfactions.md`
- Примени правила (см. выше)
- Обнови Dissatisfactions.md

#### 5. Стратегическая сверка

- Прочитай `DS-strategy/docs/Strategy.md`
- Проверь соответствие текущих РП стратегическим приоритетам
- Зафиксируй расхождения (если есть)

#### 6. Формирование плана на неделю

- Перенеси carry-over РП в новый WeekPlan
- Обнови таблицу РП (статусы, бюджеты — без изменений)
- Не добавляй новых РП

#### 7. Нерегулярные блоки (если есть в повестке)

- Проверь Session Agenda на нерегулярные блоки
- Если применимы на этой неделе — обработай

#### 8. Утверждение и синхронизация

- Смени `status: draft` → `status: confirmed` в WeekPlan
- Обнови `MEMORY.md` → секция «РП текущей недели» (актуальный список + статусы)
- Обнови `Strategy.md` (приоритеты месяца, статусы фаз)
- Очисти обработанные заметки из `fleeting-notes.md`
- **ОБЯЗАТЕЛЬНО:** Закоммить изменения в DS-strategy

**Результат:** утверждённый WeekPlan W{N} (`status: confirmed`), обновлённые Strategy.md, MEMORY.md, очищенный inbox.
